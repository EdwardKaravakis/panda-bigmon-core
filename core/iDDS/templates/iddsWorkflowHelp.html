Workflow Information:

Note that the number of workflows retrieved from the database is limited for performance reasons. By default it gets workflows in the last 7 days. If you need to increase the period, specify &days=N on the URL, but use carefully, the load on the DB is heavy for big time windows.

Workflow status

  • Finished: All tasks in the workflow have finished successfully. If tranform type is StageIn, this status is the staging status.
  • Failed: All tasks in the workflow have failed. If tranform type is StageIn, this status is the staging status.
  • Subfinished: The workflow has both finished and failed tasks. If tranform type is StageIn, this status is the staging status.
  • Cancelled: The workflow is cancelled. If tranform type is StageIn, this status is the staging status.

Columns of files

files: primary input or pseudo input datasets

  • total files: total number of primary input or pseudo input files
  • released files: number of files released to PanDA by iDDS
  • unreleased files: number of files unreleased to PanDA by iDDS
  • finished files: percentage of files PanDA has processed successfully
  • failed files: percentage of files PanDA has failed to process
